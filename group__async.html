Detailed Description

async

This is a snippet of the OpenACC code.

#pragma acc parallel loop
for (int i = 0; i < limit; ++i) {
array[i] = i;
}

By default, the CPU thread will wait/block until the parallel loop finishes. If you want the CPU thread to proceed without waiting for the parallel loop to finish, you may add the async directive,

#pragma acc parallel loop async(queue)
// or #pragma acc parallel loop async
for (int i = 0; i < limit; ++i) {
array[i] = i;
}

where queue is an optional hardwired integer or an integer variable. A special integer constant value acc_async_sync is defined in the OpenACC standard that can be used in the async directive as a queue number, to obtain an synchronous/blocking behavior. Implementations may be different on different platforms though, on the CUDA platform, every OpenACC queue is built on top of a CUDA stream.
